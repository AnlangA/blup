# Apps Module — Phase 3: Bevy Viewer

## Module Overview

`apps/bevy-viewer` is an optional interactive rendering host for 2D/3D learning scenes. It displays `SceneSpec` documents generated by Core or domain plugins. It is embedded as a view within the learning interface — not as a replacement for the Web UI.

**Core principle:** Bevy is an interactive rendering layer, not the primary UI. Text, forms, chat, and code display remain in the Web UI. Bevy renders scenes that benefit from interactive visualization: physics simulations, 3D molecule models, geometry demonstrations, data structure animations.

## Bevy Systems (Full Implementation)

### Scene Loader

```rust
// scene/loader.rs
use bevy::prelude::*;
use bevy::pbr::{DirectionalLight, PointLight, StandardMaterial};
use bevy::color::Color;

#[derive(Resource)]
struct SceneMarker;

#[derive(Component)]
struct SceneEntity;

#[derive(Component)]
struct Interactable {
    on_click: InteractionAction,
    on_hover: InteractionAction,
}

enum InteractionAction {
    Highlight,
    ShowLabel,
    Rotate,
    ZoomTo,
    None,
}

fn load_scene_system(
    mut commands: Commands,
    mut meshes: ResMut<Assets<Mesh>>,
    mut materials: ResMut<Assets<StandardMaterial>>,
    spec: Res<SceneSpec>,
    existing_scene: Query<Entity, With<SceneEntity>>,
) {
    // Clear previous scene
    for entity in existing_scene.iter() {
        commands.entity(entity).despawn_recursive();
    }

    // Set up camera
    let cam = &spec.camera;
    commands.spawn((
        Camera3d::default(),
        Transform::from_xyz(cam.position.x, cam.position.y, cam.position.z)
            .looking_at(
                Vec3::new(cam.target.x, cam.target.y, cam.target.z),
                Vec3::Y,
            ),
        SceneEntity,
    ));

    // Set up lights
    for light in &spec.lights {
        match light.light_type.as_str() {
            "directional" => {
                commands.spawn((
                    DirectionalLight {
                        color: parse_color(&light.color),
                        illuminance: 10000.0 * light.intensity,
                        ..default()
                    },
                    Transform::from_xyz(light.direction.x, light.direction.y, light.direction.z)
                        .looking_at(Vec3::ZERO, Vec3::Y),
                    SceneEntity,
                ));
            }
            "ambient" => {
                // Ambient light via environment map or clear color
            }
            "point" => {
                commands.spawn((
                    PointLight {
                        color: parse_color(&light.color),
                        intensity: 500.0 * light.intensity,
                        range: 20.0,
                        ..default()
                    },
                    Transform::from_xyz(light.direction.x, light.direction.y, light.direction.z),
                    SceneEntity,
                ));
            }
            _ => {}
        }
    }

    // Spawn entities
    for entity_spec in &spec.entities {
        let mut entity = commands.spawn((SceneEntity,));

        // -- Transform --
        entity.insert(Transform::from_xyz(
            entity_spec.position.x, entity_spec.position.y, entity_spec.position.z,
        ));

        // -- Mesh --
        if let Some(renderable) = &entity_spec.renderable {
            let mesh = match renderable.shape.as_str() {
                "sphere" => meshes.add(Sphere::new(renderable.params.get("radius")
                    .and_then(|v| v.as_f64()).unwrap_or(0.5) as f32)),
                "cube" => meshes.add(Cuboid::new(1.0, 1.0, 1.0)),
                "cylinder" => {
                    let r = renderable.params.get("radius").and_then(|v| v.as_f64()).unwrap_or(0.1) as f32;
                    let h = renderable.params.get("height").and_then(|v| v.as_f64()).unwrap_or(2.0) as f32;
                    meshes.add(Cylinder::new(r, h))
                }
                _ => meshes.add(Sphere::new(0.3)),
            };

            let color = parse_color(&renderable.color);
            let material = materials.add(StandardMaterial {
                base_color: color,
                perceptual_roughness: match renderable.material.as_deref() {
                    Some("metallic") => 0.3,
                    Some("glossy") => 0.1,
                    _ => 0.8,
                },
                metallic: if renderable.material.as_deref() == Some("metallic") { 0.8 } else { 0.0 },
                ..default()
            });

            entity.insert((Mesh3d(mesh), MeshMaterial3d(material)));
        }

        // -- Label --
        if let Some(label) = &entity_spec.label {
            entity.insert(Name::new(label.clone()));
        }

        // -- Interactable --
        if let Some(interactable) = &entity_spec.interactable {
            entity.insert(Interactable {
                on_click: parse_action(&interactable.on_click),
                on_hover: parse_action(&interactable.on_hover),
            });
        }
    }

    info!("Scene loaded: {} entities", spec.entities.len());
}
```

### Camera Controller

```rust
// scene/camera.rs
use bevy::input::mouse::{MouseMotion, MouseWheel};
use bevy::prelude::*;

#[derive(Component)]
struct OrbitCamera {
    target: Vec3,
    distance: f32,
    yaw: f32,
    pitch: f32,
    min_distance: f32,
    max_distance: f32,
    orbit_speed: f32,
    zoom_speed: f32,
}

impl Default for OrbitCamera {
    fn default() -> Self {
        Self {
            target: Vec3::ZERO,
            distance: 5.0,
            yaw: 0.0,
            pitch: 0.3,
            min_distance: 1.0,
            max_distance: 20.0,
            orbit_speed: 0.003,
            zoom_speed: 0.1,
        }
    }
}

fn orbit_camera_system(
    mut query: Query<(&mut Transform, &mut OrbitCamera)>,
    mut mouse_motion: EventReader<MouseMotion>,
    mut scroll: EventReader<MouseWheel>,
    mouse_button: Res<ButtonInput<MouseButton>>,
) {
    for (mut transform, mut cam) in query.iter_mut() {
        // Orbit: right mouse button + drag
        if mouse_button.pressed(MouseButton::Right) {
            for event in mouse_motion.read() {
                cam.yaw -= event.delta.x * cam.orbit_speed;
                cam.pitch -= event.delta.y * cam.orbit_speed;
                cam.pitch = cam.pitch.clamp(-1.5, 1.5); // Limit pitch
            }
        }

        // Zoom: scroll wheel
        for event in scroll.read() {
            cam.distance -= event.y * cam.zoom_speed;
            cam.distance = cam.distance.clamp(cam.min_distance, cam.max_distance);
        }

        // Calculate camera position from spherical coordinates
        let position = cam.target + Vec3::new(
            cam.distance * cam.pitch.cos() * cam.yaw.sin(),
            cam.distance * cam.pitch.sin(),
            cam.distance * cam.pitch.cos() * cam.yaw.cos(),
        );

        transform.translation = position;
        transform.look_at(cam.target, Vec3::Y);
    }
}
```

### Entity Picking (Click Detection)

```rust
// scene/interaction.rs
fn entity_picking_system(
    windows: Query<&Window>,
    cameras: Query<(&Camera, &GlobalTransform)>,
    mouse_button: Res<ButtonInput<MouseButton>>,
    interactables: Query<(Entity, &GlobalTransform, &Interactable)>,
    meshes: Query<&Handle<Mesh>>,
    mesh_assets: Res<Assets<Mesh>>,
    mut events: EventWriter<SceneInteractionEvent>,
) {
    if !mouse_button.just_pressed(MouseButton::Left) {
        return;
    }

    let window = windows.single();
    let (camera, camera_transform) = cameras.single();

    // Get cursor position in window coordinates
    let Some(cursor_pos) = window.cursor_position() else { return };

    // Cast ray from camera through cursor
    let Ok(ray) = camera.viewport_to_world(camera_transform, cursor_pos) else { return };

    // Check intersection with each interactable entity
    let mut closest: Option<(Entity, f32)> = None;

    for (entity, transform, interactable) in interactables.iter() {
        // Simple sphere intersection test for now
        // (Full mesh raycasting would use bevy_mod_raycast or similar)
        let entity_pos = transform.translation();
        let sphere_radius = 0.6; // From entity metadata

        if let Some(distance) = ray_intersect_sphere(&ray, entity_pos, sphere_radius) {
            if closest.map_or(true, |(_, d)| distance < d) {
                closest = Some((entity, distance));
            }
        }
    }

    if let Some((entity, _)) = closest {
        events.send(SceneInteractionEvent::EntityClicked { entity });
    }
}

fn ray_intersect_sphere(ray: &Ray3d, center: Vec3, radius: f32) -> Option<f32> {
    let oc = ray.origin - center;
    let a = ray.direction.dot(ray.direction);
    let b = 2.0 * oc.dot(*ray.direction);
    let c = oc.dot(oc) - radius * radius;
    let discriminant = b * b - 4.0 * a * c;
    if discriminant < 0.0 { return None; }
    Some((-b - discriminant.sqrt()) / (2.0 * a))
}
```

### Animation System

```rust
// scene/animation.rs
#[derive(Component)]
struct SceneAnimation {
    animation_type: AnimationType,
    axis: Vec3,
    speed: f32,
    looped: bool,
    elapsed: f32,
}

enum AnimationType {
    Rotate,
    Oscillate { amplitude: f32, frequency: f32 },
    Pulse { min_scale: f32, max_scale: f32 },
}

fn animation_system(
    mut query: Query<(&mut Transform, &mut SceneAnimation)>,
    time: Res<Time>,
) {
    for (mut transform, mut anim) in query.iter_mut() {
        anim.elapsed += time.delta_secs();

        match anim.animation_type {
            AnimationType::Rotate => {
                let angle = anim.speed * time.delta_secs();
                transform.rotate_axis(anim.axis, angle);
            }
            AnimationType::Oscillate { amplitude, frequency } => {
                let offset = amplitude * (anim.elapsed * frequency).sin();
                transform.translation += anim.axis * offset;
            }
            AnimationType::Pulse { min_scale, max_scale } => {
                let t = (anim.elapsed * 2.0).sin() * 0.5 + 0.5;
                let scale = min_scale + (max_scale - min_scale) * t;
                transform.scale = Vec3::splat(scale);
            }
        }
    }
}
```

### Scene Plugin Types (Detailed)

**Molecule Viewer** — specializations of the generic scene loader:
- CPK atom colors (H=white, C=dark gray, N=blue, O=red, S=yellow, P=orange)
- Bond rendering: cylinders between atom positions with radius proportional to bond order
- Auto-rotation toggle; clicking an atom shows its properties panel

**Physics Sim** — using `avian3d` physics engine:
- `RigidBody::Dynamic` for moving objects, `RigidBody::Static` for ground/surfaces
- `Collider` components from mesh shapes
- `Gravity` resource configurable from SceneSpec
- Force application from SceneSpec `forces` array

**Geometry Explorer** — 2D shapes in 3D space:
- Points rendered as small spheres; lines as thin cylinders; polygons as extruded shapes
- Step-through construction: each SceneSpec entity has an `order` value
- Coordinate axes and grid rendered as reference

**Graph Visualizer** — force-directed layout:
- Nodes as spheres with labels; edges as line strips
- Simple Fruchterman-Reingold force simulation in `Update` system
- Traversal animation: highlights nodes/edges in sequence

## Embedding Strategy

### Option A: Off-Screen Render + Compositing (Recommended)

Bevy renders to an off-screen texture. The texture is shared with the Tauri webview via GPU memory (e.g., IOSurface on macOS, DirectX texture sharing on Windows). The webview displays it in a `<canvas>` or swaps buffers.

**Pros:** Bevy and webview share the same window; seamless UX.
**Cons:** Cross-platform GPU sharing is complex; requires significant Tauri integration work.

### Option B: Separate Window

Bevy runs in its own native window. Tauri manages window positioning.

**Pros:** Simple to implement; works cross-platform.
**Cons:** Two windows feel less integrated; window management complexity.

### Option C: Render to Web via WASM

Bevy compiles to WASM and renders to a `<canvas>` in the webview directly.

**Pros:** True embedding in webview; no native window complexity.
**Cons:** WASM performance limitations; no GPU compute; Bevy WASM support is experimental. Large binary size.

**Recommendation for Phase 3:** Start with Option B (separate window) for feasibility. Prototype Option A. Re-evaluate Option C when Bevy's WASM support matures.

## Risk Register

| Risk | Impact | Mitigation |
|------|--------|------------|
| GPU texture sharing (Option A) not portable | Can't embed Bevy in webview | Start with separate window (Option B); prototype sharing on each platform |
| Bevy scene loading is slow for complex scenes | Poor UX, frame drops | Scene loading on background thread; progressive loading for large scenes |
| Bevy version churn | Frequent breaking changes | Pin Bevy version; abstract scene loading behind trait |
| WASM performance if Option C pursued | Low FPS, large download | Only use WASM for simple 2D scenes; native for 3D |
| Plugin Physics library compatibility | avian/bevy_rapier may lag Bevy versions | Keep physics behind feature flag; core viewer works without it |
