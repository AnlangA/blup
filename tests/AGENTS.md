# AGENTS.md

## 1. 目标

`tests` 存放跨模块测试、集成测试、端到端测试、插件契约测试、沙箱安全测试和学习流程回归测试。

## 2. 目标实现的路径

- 为 Agent 状态机设计流程测试。
- 为 schema 设计兼容性和校验测试。
- 为插件接口设计 contract test。
- 为沙箱设计超时、资源限制、恶意输入测试。
- 为 UI 和 Bevy 交互设计端到端测试。

## 3. 需要联网查找/参考的资料与核心思想

需要查找：

- Rust cargo test、insta snapshot、proptest 文档。
- Playwright 或 WebDriver 测试资料。
- Tauri 测试资料。
- Bevy headless 或场景测试资料。
- 沙箱安全测试案例。

核心思想：

- 学习流程必须可回归验证。
- 插件必须通过契约测试才能接入。
- 沙箱必须优先测试失败、超时和恶意输入。
- LLM 相关测试应使用固定响应或 mock，避免不稳定。

## 4. 不允许做什么事情

- 不允许测试依赖真实用户隐私数据。
- 不允许在默认测试中调用付费 LLM API。
- 不允许测试裸跑不可信代码。
- 不允许忽略沙箱失败、超时或非零退出码。
